# HRManager
Початкова сторінка сайту (Start page)
У верхній правій частині екрану розміщений логотип (назва додатку) з гіперпосиланням на головну сторінку сайту.

Сторінка входу (Log-in page)
У верхній лівій частині екрану знаходиться кнопка авторизації користувача, якщо користувач не зареєстрований то зі сторінки авторизації можне перейти на реєстраційну сторінку.

Нижче розташоване навігаційне меню по сайту.

В самому низу сторінки знаходиться футер, який не несе ніякої функціональності окрім, як естетичного вигляду.

З головної сторінки користувач має можливість потрапити на сторінку авторизації, або на сторінку реєстрації. Вони обидві між собою схожі та інтуїтивно зрозумілі і призводять до входу на сайт, як авторизований користувач.

Авторизувавшись користувач має можливість потрапити на сторінки:
Personal Info
Edit
Requests

При переході на сторінку Home у користувача відкривається сторінка з таблицею в якій відображається особиста інформація:
Ім’я
Електронна пошта
Пароль
Команда до якої він відноситься
Кількість днів, що залишилися по хворобі (на цей рік)
Кількість вихідних днів що залишилися (на цей рік)

Якщо перейти на сторінку Edit у користувача відкриється сторінка редагування особистої інформації (ім’я, електронної пошти та паролю).

На сторінці Requests перед користувачем постане сторінка з таблицею в якій будуть записані всі запити на відпустку з такими полями:
Тип відпустки (лікарняні, звичайні вихідні)
Дата початку
Дата закінчення
Статус (прийнято, відхилено, розглядається).

І останній сторінці New request користувачу запропонують оформити нову заявку на відпустку.
